Alternate Flow - Deferred Processing

This flow represents the Deferred Processing path, allowing users to quickly capture receipts and complete details later:

Initial Screen

Start by selecting expense entry method

View Screen →

Receipt Capture

Take photo and select "Save Draft"

View Screen →

Draft Saved

Quick confirmation, return to previous context

→

Expense List (UC-EL)

Later, access draft in expense list

→

Complete Draft

Continue normal processing flow

Deferred Processing: The Deferred Processing workflow enables users to quickly capture receipt images and save them as Draft Expenses for later completion. This provides flexibility for users who need to capture receipts quickly without immediately processing all details. Draft Expenses can later be accessed from the Expense List (UC-EL) for completion.
Conditional Feature: The Tax Details screen only appears when the user has enabled tax features via the Tax Registration Status setting in their account profile. This is a conditional module that adapts to the business's registration status.

Implementation Notes

  • These mockups demonstrate a mobile-first approach for the expense tracking workflow
  • OCR functionality requires camera access and may require connectivity for processing
  • Job suggestions leverage contextual data (location, calendar, recent activity)
  • Core expense tracking functionality works offline with data syncing when connectivity returns
  • Deferred Processing workflow allows quick capture of receipt images as Draft Expenses
  • Draft Expenses can later be completed through the Expense List interface (UC-EL)
  • Navigation is contextual, adapting to the user's workflow and entry point
  • The Tax Details screen is conditional based on the user's Tax Registration Status
  • The user interface emphasizes ease of use, with clear visual feedback and intuitive controls
